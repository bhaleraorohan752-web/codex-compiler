Understanding the CodeX Cloud Infrastructure.

BACKEND

Isolated Execution (Sandboxing)

Every time you click Run, CodeX triggers a secure workflow. Your code is not executed on our main server. Instead, it is isolated inside a Docker Container.

  • Resource Limits: Containers are capped at 128MB RAM to prevent memory leaks.
  • Network Isolation: Outbound network access is disabled for user code.
  • Auto-Cleanup: Containers and temporary files are destroyed immediately after execution.
NETWORKING

The Interactive Terminal

CodeX uses WebSockets (Socket.io) to handle the communication between your browser and the Linux environment. Unlike traditional compilers that wait for the code to finish, CodeX streams data line-by-line.

This allows for scanf() in C or input() in Python to work just like a local terminal.
